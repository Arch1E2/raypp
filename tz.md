# 

Задача: Мини-сервис "Умный FAQ для ********"

Цель: Создать backend-сервис, который обрабатывает вопросы пользователей и отвечает на них, используя базу знаний компании.

Что нужно реализовать:

**API-сервис (FastAPI/Flask)**

POST /api/ask - принимает текстовый вопрос, возвращает ответ от LLM
POST /api/documents - загрузка документов в базу знаний (txt/md)
GET /api/health - healthcheck

**RAG Pipeline**

Загрузка 3-5 документов в векторную БД (можно использовать ChromaDB/Qdrant/pgvector)
Поиск релевантных фрагментов по вопросу
Генерация ответа через Anthropic/OpenAI API с использованием найденного контекста

**Кэширование**

Redis для кэширования частых вопросов (TTL 1 час)

**База данных**

PostgreSQL для хранения истории запросов (вопрос, ответ, timestamp, токены)

**Простой веб-интерфейс**

Одна HTML-страница с формой для вопросов
Показ ответа и использованных источников

**Docker**

docker-compose.yml с сервисами: API, PostgreSQL, Redis, векторная БД
Всё должно запускаться одной командой

**Тесты**

Минимум 3 unit-теста (pytest)
1 интеграционный тест API endpoint

**Критерии оценки:**
**Обязательно:**

Работающий сервис из Docker Compose
Интеграция с LLM API
RAG с векторным поиском
Кэширование в Redis
README с инструкцией запуска

Использование Claude Code/AI в разработке (Нужны будут скриншоты)
Логирование
Метрики (подсчет токенов, время ответа)
Eval для проверки качества ответов

**Будет плохо если:**

Не запускается с первого раза
Нет обработки ошибок
Хардкод API ключей в коде
Отсутствие .env.example
